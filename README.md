# YLBooks — 말로 만드는 내 키링 리듬게임

> 바이브코딩(AI에게 말로 시켜 만드는 코딩) **교재 집필 저장소**입니다.
> 컴퓨터가 낯선 완전 초보(60~70대 포함)도 **AI에게 프롬프트만 건네** 리듬게임을 만들고,
> **키링 키보드**(블루투스·방향키)로 직접 플레이하게 안내하는 책을 만듭니다.

---

## 🤖 이 저장소를 AI에게 파악시키려면 (제일 먼저 읽을 것)

이 프로젝트를 이어서 작업하거나 이해해야 한다면, AI(예: Claude Code)에게 이렇게 말하세요:

> **"이 저장소의 `CLAUDE.md`(또는 `AGENTS.md`)를 먼저 읽고 프로젝트를 파악해줘."**

- `CLAUDE.md` / `AGENTS.md` — **AI 온보딩 안내서.** 프로젝트가 무엇이고, 무엇을 지켜야 하며(예: "코드는 본문에 안 보여준다"), 파일이 어디에 있고, 어떻게 빌드하고, 다음에 뭘 하면 되는지를 순서대로 정리했습니다. 두 파일 내용은 같습니다(도구에 따라 읽는 파일이 달라서 둘 다 둡니다).

비전공자라도, 위 한 문장만 AI에게 전하면 AI가 알아서 구조를 파악합니다.

---

## 이 책은 무엇인가

- **주제**: 바이브코딩으로 **리듬게임**을 만들고, **키링 키보드**로 플레이하기.
- **대상 독자**: 컴퓨터·코딩을 처음 접하는 **완전 초보**(전공·영어·수학 전제 없음).
- **원칙**: 본문에 **소스코드를 싣지 않습니다.** 독자는 AI에게 건네는 **프롬프트(말)**와, **실제 화면 캡처**만 보고 한 단계씩 따라 합니다.

## 책 구성 (6마당 · 21차시 · 부록 A~D)

| 마당 | 차시 | 내용 |
|---|---|---|
| 0 들어가기 | 서문 | 이 책의 약속·읽는 법 |
| 1 바이브코딩 첫 만남 | 1~2 | 바이브코딩이란 · 도구 지도 |
| 2 컴퓨터 기초 | 3~5 | 터미널 · 파일 탐색기 · Node.js 설치 |
| 3 AI 도구 설치 | 6~10 | Claude Code(주력)·길들이기 · Codex · OpenCode · Antigravity |
| 4 리듬게임 만들기 | 11~16 | 첫 프로젝트 · 노트 · 판정 · 점수 · 음악 · 꾸미기 (Expo Web) |
| 5 키링으로 플레이 | 17~19 | 블루투스 페어링 · 플레이 · 트러블슈팅 |
| 6 자랑·마무리 | 20~21 | 공유 · 다음 단계 |
| 부록 | A~D | 용어 사전 · 계정·요금 · 오류 해결 · 프롬프트 모음 |

## 저장소 구조

```
YLBooks/
├── README.md              ← (이 파일) 사람용 안내
├── CLAUDE.md / AGENTS.md  ← AI 온보딩 안내서 (제일 먼저 읽히게 하는 파일)
├── .gitignore
└── book2/                 ← 책 본체
    ├── PLAN.md            전체 기획안 (목차·차시표·제작 방식·리스크)
    ├── CONTEXT.md         확정 사양·결정·용어 (프로젝트의 "진실" 문서)
    ├── FORMAT.md          집필 규격 (차시 템플릿·박스 문법·이미지 규약)
    ├── CAPTURE_QUEUE.md   그림 현황 + 아직 못 찍은 "저자 촬영" 목록
    ├── 리듬게임교재_전체_v3.docx   ← 완성본(읽기용, 합본)
    ├── lessons/           원고(집필 소스, 마크다운) — 00_intro … 21_roadmap, A~D
    ├── images/            본문 그림
    │   ├── game/          게임 실제 실행 캡처 (시작·난이도·플레이)
    │   ├── c02/           공식 사이트 캡처 (Claude Code·Codex·OpenCode·Node·Expo)
    │   ├── c03/ c05/ c06/ 실제 터미널 창 캡처
    │   └── …
    ├── specs/             캡처 스펙(JSON)
    └── tools/             빌드·캡처 스크립트 (아래 참조)
```

> 본문 편집은 **항상 `book2/lessons/*.md`** 에서 합니다. `.docx`는 빌드로 재생성되는 산출물이라 직접 고치지 않습니다.

## 완성본 보기

가장 쉬운 방법: **`book2/리듬게임교재_전체_v3.docx`** 를 워드로 엽니다. (서문+21차시+부록, 그림 포함 합본)

## 원고를 고친 뒤 다시 빌드하기 (관리)

1. **환경 준비**(한 번만): 파이썬 설치 후
   ```
   pip install python-docx pillow
   ```
2. **원고 수정**: `book2/lessons/`의 해당 `.md` 파일 편집.
3. **부록 D(프롬프트 모음) 갱신**(프롬프트를 바꿨다면):
   ```
   python book2/tools/build_appendix_d.py
   ```
4. **합본 docx 재생성**:
   ```
   python book2/tools/build_docx.py
   ```
   → `book2/리듬게임교재_전체_v3.docx` 가 새로 만들어집니다.

> ⚠️ 워드에서 그 docx를 **열어 둔 채로 빌드하면 저장이 막힙니다.** 워드를 닫고 실행하세요.

## 이미지(그림) 방침 — 중요

- **실제 캡처 우선.** 목업·클립아트가 아니라 진짜 화면을 찍습니다(스크린샷 80%+ 목표).
- **개인정보 안전 규칙**: 전체 화면 캡처는 **금지**(다른 창이 함께 찍혀 유출됨). 반드시 **창 하나만** 잡는 `book2/tools/winshot.py`(Windows `PrintWindow`)를 씁니다.
- **캡처 도구**(`book2/tools/`):
  - `winshot.py` — 지정한 창 하나만 캡처(개인정보 안전).
  - `cap_terminal.py` — 실제 파워셸(conhost) 창 캡처 + 여백 자동 크롭.
  - `capture_game.py` — Expo 리듬게임을 실제 실행해 게임 화면 캡처(Playwright 필요).
- 아직 못 찍어 **저자가 직접 촬영해야 하는 그림**(설치 마법사·블루투스 페어링·물리 키링 사진 등)은 `book2/CAPTURE_QUEUE.md`에 이유·참고 소스와 함께 정리돼 있습니다.

## 현재 상태

- ✅ **원고**: 서문+21차시+부록 A~D 초안 완성. 본문 소스코드 노출 0건.
- ✅ **그림**: 게임 5컷 + 공식 사이트 5컷 + 실제 터미널 4컷 = **실제 캡처 14종** 반영.
- ⏳ **남은 그림**: 설치 마법사·파일 탐색기 창·블루투스 설정·물리 키링 사진 등은 **저자 촬영 대기**(`CAPTURE_QUEUE.md`).
