# CLAUDE.md — AI 온보딩 안내서

이 저장소에서 작업하는 **AI 에이전트가 가장 먼저 읽는 파일**입니다. 사람이 "이 프로젝트를 파악해줘 / 이어서 작업해줘"라고 하면, 아래 순서대로 읽고 이 규칙을 지키세요. (내용은 `AGENTS.md`와 동일합니다.)

## 0. 한 줄 요약
바이브코딩으로 **키링(블루투스·방향키) 리듬게임**을 **완전 왕초보**가 **프롬프트만으로** 만드는 **한국어 교재**를 집필하는 저장소. 책 본체는 `book2/`.

## 1. 먼저 읽을 순서
1. `book2/CONTEXT.md` — **확정 사양·결정·용어(프로젝트의 진실 문서).** 여기와 충돌하는 서술을 만들지 말 것.
2. `book2/PLAN.md` — 전체 목차·차시표·제작 방식·리스크.
3. `book2/FORMAT.md` — 집필 규격(차시 템플릿, 박스 문법, 이미지 규약).
4. `book2/CAPTURE_QUEUE.md` — 그림 현황과 "저자 촬영 대기" 목록.
5. `book2/lessons/06_claudecode.md`(또는 아무 차시 1개) — **문체·구조 견본.** 새 원고는 이 톤을 따를 것.

## 2. 절대 지킬 원칙 (hard rules)
- **본문에 소스코드를 노출하지 않는다.** 코드블록은 오직 ①🗣️ 프롬프트(AI에게 하는 말) ②터미널에 입력하는 명령/화면 안내. `.js .py .ino` 등 프로그램 코드 금지.
- **대상 = 완전 왕초보(60~70대 포함).** 존댓말, 짧은 문장, 전문용어는 처음 나올 때 괄호로 즉시 풀이, **한 단계 = 한 그림**.
- **변동 정보(요금·모델명·버전·SDK)를 본문에 못박지 않는다** → "공식 사이트 확인 / 대략"으로. (구버전을 현행처럼 쓰는 것이 최악의 실수.)
- **FORMAT.md의 차시 템플릿을 그대로 따른다**(note→왜→단계①②③(각 FIG)→tip/warning→success→abstract→question→info 낱말사전→다음 차시 예고).

## 3. 확정 기술 사양 (이대로만 서술 — 바꿔 쓰지 말 것)
- **컨트롤러 = 키링 키보드 = 블루투스 무선 · 방향키 4개(← ↓ ↑ →).** USB/유선/DFJK 아님. 페어링만 하면 키보드로 인식. 예시 실물 = `nordic_keyboard`.
- **게임 = Expo Web(브라우저).** 실행 `npx expo start --web`. 브라우저 `keydown`(Arrow*)로 조작. (네이티브 앱 아님.)
- **노트 자동 채보(오디오 분석 자동생성) 없음** → 미리 정해둔 고정 노트 패턴(120BPM).
- **도구 4종**: **Claude Code(주력)** / Codex(챗GPT 계정 로그인) / OpenCode(오픈소스·`/connect`) / Antigravity(구글·GUI). 핵심 메시지 "프롬프트는 어느 도구든 똑같이 통한다".
- 독자는 코드를 안 짜고 **프롬프트만 입력**, AI가 만든 결과를 **실행·확인**만 한다.
- 참고용 예시 게임 소스(이 저장소 밖): `C:\Users\user\keyboard-game\rhythm` (Expo/React Native).

## 4. 파일 지도
- 원고(편집은 **여기서만**): `book2/lessons/*.md` (`00_intro`…`21_roadmap`, 부록 `A_glossary`·`B_accounts`·`C_errors`·`D_prompts`)
- 그림: `book2/images/` — `game/`(게임 실행 캡처)·`c02/`(공식 사이트)·`c03,c05,c06/`(터미널)
- 완성본(빌드 산출물, **직접 편집 X**): `book2/리듬게임교재_전체_v3.docx`
- 빌드·캡처 스크립트: `book2/tools/`

## 5. 빌드·관리 (how to build)
```
pip install python-docx pillow          # 최초 1회
python book2/tools/build_appendix_d.py  # 프롬프트 바꿨으면: 부록 D 재수집
python book2/tools/build_docx.py        # 합본 docx 재생성 (워드 닫고 실행)
```
경로는 스크립트 위치(`__file__`) 기준이라 어디서 실행해도 동작.

## 6. 그림 캡처 규칙 (개인정보 안전 — 반드시 준수)
- **전체 화면 캡처 금지.** 이 PC엔 다른 창이 떠 있어 유출됨. 반드시 **창 하나만** 잡는 `book2/tools/winshot.py`(Windows `PrintWindow`) 사용.
- Win11 **파일 탐색기·설정**은 WinUI라 PrintWindow가 검게 나옴 → **저자 수동 촬영**.
- **블루투스 설정·로그인 화면** 등 개인정보가 보이는 창은 자동 캡처 **제외**, `CAPTURE_QUEUE.md`에 저자 촬영으로 남김.
- 도구: `winshot.py`(창 단위)·`cap_terminal.py`(파워셸 창+크롭)·`capture_game.py`(Expo 게임 실행 캡처, Playwright 필요).
